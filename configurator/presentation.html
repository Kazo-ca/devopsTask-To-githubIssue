Le Problème Actuel

Silos de données

La gestion de projet est dans Azure DevOps, mais le code et l'IA sont dans GitHub.

Perte de temps

Copier-coller manuel des descriptions et des consignes entre les plateformes.

Traçabilité fragile

Difficulté à lier un commit ou une Pull Request à une tâche spécifique.

IA hors-contexte

Les agents d'IA (Copilot) n'ont pas accès aux détails de la User Story parente.

Le rôle du Développeur

Contrôle avant exécution

Le développeur vérifie que l'issue générée contient le bon scope. L'agent ne peut pas démarrer sans une interaction humaine sur GitHub.

Contrôle après Pull Request

Une fois la PR créée (après l'auto-revue de l'IA), le développeur effectue la revue finale. Il peut demander des ajustements à l'IA via des commentaires.

"La technologie accélère le code, l'humain garantit la valeur métier."
