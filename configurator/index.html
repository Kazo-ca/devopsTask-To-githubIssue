🔐 Aide – Création des secrets GitHub

        

            KazoAdoSync requiert deux Personal Access Tokens (PAT) stockés comme secrets dans votre dépôt GitHub.
        

        

            

                
1. Créer ADO_PAT_TOKEN

                
    
                    
  1. Accédez à Azure DevOps.
    2. 
                    
  2. Cliquez sur votre avatar → Personal access tokens.
    3. 
                    
  3. Cliquez New Token et activez les scopes :
    
                        Work Items – Read & Write
                    
    4. 
                    
  4. Copiez le token généré.
    5. 
                    
  5. Dans votre dépôt GitHub : Settings → Secrets and variables → Actions → New repository secret.
    6. 
                    
  6. Nom : ADO_PAT_TOKEN — collez la valeur.
    7. 
                

            

            

                
2. Créer GH_PAT_TOKEN

                
    
                    
  1. Accédez à GitHub → Settings → Developer settings → Fine-grained tokens.
    2. 
                    
  2. Cliquez Generate new token.
    3. 
                    
  3. Activez les permissions :
    
                        Issues – Read & Write
                    
    4. 
                    
  4. Copiez le token généré.
    5. 
                    
  5. Dans votre dépôt GitHub : Settings → Secrets and variables → Actions → New repository secret.
    6. 
                    
  6. Nom : GH_PAT_TOKEN — collez la valeur.
    7. 
                

            

        

        

            ⚠️ Sécurité : Ne partagez jamais vos tokens. Utilisez toujours les GitHub Secrets pour les stocker, jamais en clair dans vos fichiers de configuration.
